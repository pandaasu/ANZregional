This function facilitates the creation of new jobs in the Interface Control System. The following job types can be created:

Job Identifier: (Mandatory) A globally unique job identifier

Job Description: (Mandatory) A description of the job

Job Resource Group: (Optional) The job resource group must match an Oracle consumer group when entered. An Oracle consumer group allows the DBA team to assign specific server resources to the job

Job Execution History: (Mandatory) A number of job execution traces to be retained in the database

Job Operator Alert Flag: (Mandatory) A flag indicating operator alerts are to be raised for execution exceptions

Job Email Group: (Optional) An email group for emails to be sent for execution exceptions

Job Type: (Mandatory) The job type - *INBOUND (interfaces coming into ICS), *OUTBOUND (interface going out from ICS), *PASSTHRU (interfaces passing through ICS), *DAEMON (background server job), *PROCEDURE (scheduled procedure)

Job Group: (Optional) A group identifier for server jobs (*INBOUND, *OUTBOUND, *PASSTHRU, *DAEMON). Server jobs process interfaces or tasks belonging to the same group)

Job Procedure: (Optional) The Oracle procedure that will be executed by *DAEMON or *PROCEDURE

Job Next: (Mandatory) The next execution date for the job (server jobs *INBOUND, *OUTBOUND, *PASSTHRU, *DAEMON must be Oracle SYSDATE). The expression can be any valid Oracle expression that resolves to a date.

Job Interval: (Optional) The interval between execution dates for the job (server jobs *INBOUND, *OUTBOUND, *PASSTHRU, *DAEMON must not be specified). The expression can be any valid Oracle expression that resolves to a date.

Job Status: (Mandatory) A flag indicating the status of the job
